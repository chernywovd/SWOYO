Программа представляет собой сервис для отправки SMS через HTTP-запросы. Она принимает данные двух номеров телефона и непосредственно смс из командной строки. Далее формирует HTTP-запрос к внешнему сервису отправки SMS и выводит результат выполнения запроса. 

Для начала работы программы необходимо запустить мок-сервер:
Linux: ./prism-cli-linux mock sms-platform.yaml
macOS: ./prism-cli-macos mock sms-platform.yaml
Windows: ./prism-cli-win.exe mock sms-platform.yaml

Далее прописываем номера телефонов отправителя, получателя и текст смс сообщения.
Пример:
./python main.py --sender +79875543202 --recipient 89875543211 --message 'Привет! Лови смс'

Номера должны быть корректными, а сообщение иметь не более 1000 символов, в скрпите прописана функция проверки.

Далее программа формирует и отправляет HTTP-запрос, производит логирование и возвращает код ответа.

В моём случае программа возвращает код ответа 400. Вероятно ошибка связана с авторизаций пользователя на мок-сервере, так как введённые данные парсятся корректные, а методы отрабатывают как должны (проверялось через дебаг).
